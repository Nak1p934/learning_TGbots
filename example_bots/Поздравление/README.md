Этот бот позволяет вам эфектно поздравить кого-то
# Туториал по настройке бота
Выполните `git clone https://github.com/Nak1p934/learning_TGbots`
Перейдите в example_bots/Поздравление/README.md и пройдите все шаги
# Работа с bot father
Откройте Телеграм и в поиски найдите бота `@BotFather`
Откройти мини апку и создайте своего бота, затем скопируйте токен. ВНИМАНИЕ никому не показывайте токен.
Откройте файл `BOT_TOKEN`, вставьте вместо YOUR_TOKEN свой токен
# Запуск
## Первый запуск
Перейдите в `run.py` и откройте терминал.
Вам нужно активировать и создать виртуальное окружение.
1. Создайте venv - в терминале выполните `python -m venv venv`, если у вас не сработало выполните `python3 -m venv venv`
2. Активируйте venv - в терминале выполните `venv\Scripts\Activate.ps1`, если не сработало - `.\venv\Scripts\Activate.bat`
3. Установите зависимости - в терминале выполните `pip install -r requirements.txt` и дождитесь окончания загрузки
4. Запустите бота - в терминале выполните `python run.py`, если не сработало используйте `python3 run.py`
Вот и всё, бот запущен! Что бы завершить работу в терминале выполните сочитание клавиш ctrl + C.
## Советы
1. При повторном включении бота не нужно создавать venv и устанавливать зависимости повторно. Просто активируйте виртуальное окружение и запустите бота (Шаги 2 и 4)
2. Убедитесь что пользователь прежде писал боту и что пользователь не заблакировале его. В противном случае ничего не сработает
3. Вы можете проверить работу бота указав в user_spam_id свой id
# Функцианал
## Структура проекта
- Файл run.py - через него запускается проект
- Файл BOT_TOKEN - содержит переменную с токеном
- ФАЙЛ routes.py - роутеры бота
- requirements - список зависимостей
## Как использовать
Для начала вам нужно узнать id пользователя которого вы хотите поздравить.
Сделать это можно двумя способами.
1. Перешлите любое сообщение от пользователя в `@userinfobot` и он отправит вам id если у пользователя отключена пересылка попросите у него написать /start этому боту и переслать сообщение
2. Перейдите в файл `routes.py` и разкоментируйте хэндлер с обработкой старта (Он там один закоментированный не перепутать), просто уберите "#". Запустите бота. Попросите пользователя отправить вашему боту /start. Так мы убьём сразу двух зайцев узнаем id и заставим пользователя написать боту
---
После того как вы узнали id зайдите в файл routes.py и в седьмой строке (Она выглядит так - '#user_spam_id = ID') замените ID на на id пользователя, а так же раскоментируйте строчку
После этого в нужный момент отправьте боту `/spam` и ждите реакцию
